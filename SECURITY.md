# Security Policy

## Reporting a vulnerability

If you discover a security issue in the REDAX Hub specification or (post-launch) the deployed program:

1. Do not open a public issue.
2. Email security@redaxhub.com with details.
3. Allow up to 7 days for initial response.

## Audit status

REDAX Hub is not yet audited. The specification is in freeze and production code remains private until audit completion.

## Responsible disclosure

We will publicly credit responsible reporters in release notes or audit reports (with permission).
